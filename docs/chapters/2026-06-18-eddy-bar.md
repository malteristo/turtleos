# Chapter: Standing Eddy Bar (slice 2.7)

**Date:** 2026-06-18  
**Status:** Implemented locally — dogfood pending on Mac Mini  
**UX doc:** [docs/ux/README.md](../ux/README.md)

## Problem

The pinned **Eddy Door** (2026-06-16) solved blank eddy entry but lived in Discord’s **pinned messages panel** — not in the visible timeline. Practitioners could miss it. Per-message Materialize buttons cluttered the river and tied spawn to specific drops.

## Solution

**Standing eddy bar** — always the **last message** in the river channel:

| Control | Behavior |
|---------|----------|
| **new eddy** | Delete bar → spawn thread via anchor message → Discord native thread card → repost bar below |
| **flow menu** | Edit bar to flow select (+ Back) → spawn with `flow_id` → repost bar |

Thread opens as **`new eddy`**; first in-eddy message renames (unchanged). No custom thread-card embed — use Discord’s default.

After each practitioner river message, `ensure_bar_at_bottom` keeps the bar last.

Startup removes legacy pinned Eddy Door messages.

## Practitioner journey

```
#river timeline:
  … drops and thread cards …
  [ 🌀 new eddy ] [ flow menu ]
    → click new eddy
    → thread card appears, bar returns below
    → enter thread, speak, rename, Turtle joins
```
